# 区域多源固废循环利用智慧决策平台项目进展汇报

## 一、项目概述

本项目为**区域多源固废循环利用智慧决策平台**，以垃圾焚烧发电厂渗沥液处理系统为核心，建设一套集**工况感知、数据驱动、智能决策、闭环控制**于一体的智慧化管理平台。

项目目标：

- 水质稳定达标
- 设备高效运行
- 异常快速响应



## 二、需求分析

### 功能需求

#### 1）监控与分析

- 大屏总览：工艺、水质、设备、电耗、实时参数、AI 预测 vs 实际值
- 自动报表：日 / 周 / 月 / 年报，可导出上报
- 趋势分析：水质、水量、去除率、DO、电耗、吨水能耗、报警统计
- 历史回溯：多曲线对比、查任意时间点数据

#### 2）预警与诊断

- 预警范围：水质超标、设备故障、能耗异常、参数偏离、液位超限
- 分级响应：一般 / 严重 / 紧急，**≤5 分钟**声光 + 推送
- 自动诊断：给原因 + 处置建议，减少人工瞎猜

#### 3）智能优化控制

- 出水达标控制：自动协同参数，保证稳定达标
- 曝气优化：根据 DO/ORP/ 负荷调风量，**降电耗**
- 污泥优化：按需排泥，不浪费、不乱排
- 自学习：定期 / 手动让模型自己学数据，越来越准
- 目标寻优：可选水质、电耗、费用，系统自动优化

#### 4）权限与安全

- 分级登录、加密传输、3 次错码要验证码
- 密码找回：邮箱 + 30 分钟有效链接
- 操作留痕、可追溯、**等保三级**

#### 5）集成开放

- 对接：PLC、DCS、SCADA、ECS、恩菲工业互联网
- 开放 API：支持二次开发、第三方接入

------

### 技术需求

- 语言：**Python**开发智能算法
- 算法：机理模型 + AI（堆栈集成），**独立服务器部署**  【**基于污水处理生化反应原理建立的数学模型，用来精准计算水质变化，保证结果稳定可靠；再结合 AI 模型，提升预测精度和优化能力。**】
- 数据：采集有效率≥95%、延迟≤3 秒、30 秒 / 次采样
- 存储：实时≥3 年、历史≥5 年
- 性能：按钮响应≤200ms、并发≥100 人、可用性≥99.9%
- 安全：等保三级、加密、留痕、可追溯

### 2. 技术方案

系统采用前后端分离架构设计。

整体架构规划如下：

```text
PLC/SCADA/DCS/传感器
          │
          ▼
     数据采集服务
          │
          ▼
      数据处理层
    （清洗、修补）
          │
          ▼
    	数据库
          │
          ▼
     AI算法服务
      （Python）
          │
          ▼
    SpringBoot平台
          │
          ▼
        Vue大屏
```

技术栈规划：系统整体采用前后端分离架构，业务平台负责权限、数据管理和可视化，算法模块采用 Python 独立部署，通过接口与业务平台解耦。

前端：

- Vue3

后端：

- SpringBoot
- Spring Security
- MyBatis Plus
- MySQL

算法服务：

- 暂定

部署：

- Docker
